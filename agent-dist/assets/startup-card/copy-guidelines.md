# Startup Card Copy Guidelines

Use this guide to keep onboarding cards clear and consistent across hosts.

## Tone Levels

- `info`
  - Use for optional setup or neutral reminders
- `warning`
  - Use when the agent cannot safely purchase yet
- `success`
  - Use when onboarding is complete and the agent can move forward

## Title Rules

- Keep titles to 2-4 words
- Prefer imperative phrasing
- Recommended titles:
  - `Create Wallet`
  - `Fund Wallet`
  - `Continue to Offers`

## Summary Rules

- One sentence preferred
- If a checklist is critical, append compact numbered hints
- Good:
  - `Fund the wallet before buying. [1] Add TRX [2] Add USDT`
- Avoid:
  - long paragraphs
  - protocol jargon
  - multiple unrelated actions in the same sentence

## CTA Rules

- Primary CTA should match the next required action
- Secondary CTA should open details, not duplicate the primary action

## Resource Rules

- Put the most actionable resource first
- Prefer labels like:
  - `Testnet Faucet`
  - `Funding Guide`
  - `Wallet Setup Report`
